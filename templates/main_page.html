Это моё маленькое REST API. Ниже будут представлены все возможные HTTP-запросы и их функционал.

Далее под "session_id" будем понимать пользовательский ввод, корректный вид которого следующий: от u10001 до u400000 (ВКЛЮЧИТЕЛЬНО).

  • http://localhost:5000/help - Возвращает список возможных HTTP-запросов.
  • http://localhost:5000/default/info/avg - Выводит сообщение о том, чтобы пользователь заменил "/default" на нужный ему "session_id".
  • http://localhost:5000/default/info/percent - Выводит сообщение о том, чтобы пользователь заменил "/default" на нужный ему "session_id".
  • http://localhost:5000/"session_id"/info/avg - Возвращает список словарей, который содержит количество просмотров пользователя в каждой категории в абсолютных велечинах.
  • http://localhost:5000/"session_id"/info/percent - Возвращает список словарей, который содержит количество просмотров пользователя в каждой категории в процентном соотношении.
  • http://localhost:5000/"session_id"/predict - Прогнозирование пола пользователя по "session_id".
  • http://localhost:5000/"session_id"/predict/avg" - Прогнозирование пола пользователя по "session_id" с информацией о сайтах, которые он посещал в абсолютных значениях.
  • http://localhost:5000/"session_id"/predict/percent" - Прогнозирование пола пользователя по "session_id" с информацией о сайтах, которые он посещал в процентном соотношении.
