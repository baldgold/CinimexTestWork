Это моё маленькое REST API. Ниже будут представлены все возможные HTTP-запросы и их функционал.

Далее под "session_id" будем понимать пользовательский ввод, корректный вид которого следующий: от u10001 до u400000 (ВКЛЮЧИТЕЛЬНО).

  • http://localhost:5000/help - Возвращает список возможных HTTP-запросов.
  • http://localhost:5000/info/avg/default - Выводит сообщение о том, чтобы пользователь заменил "/default" на нужный ему "session_id".
  • http://localhost:5000/info/percent/default - Выводит сообщение о том, чтобы пользователь заменил "/default" на нужный ему "session_id".
  • http://localhost:5000/info/avg/"session_id" - Возвращает список словарей, который содержит количество просмотров пользователя в каждой категории в абсолютных велечинах.
  • http://localhost:5000/info/percent/"session_id" - Возвращает список словарей, который содержит количество просмотров пользователя в каждой категории в процентном соотношении.
  • http://localhost:5000/predict/"session_id" - Прогнозирование пола пользователя по "session_id".
  • http://localhost:5000/predict/avg/"session_id" - Прогнозирование пола пользователя по "session_id" с информацией о сайтах, которые он посещал в абсолютных значениях.
  • http://localhost:5000/predict/percent/"session_id" - Прогнозирование пола пользователя по "session_id" с информацией о сайтах, которые он посещал в процентном соотношении.
